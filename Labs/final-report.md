# Фінальний звіт

- Markdown Circus Docs (MCD) — це інструмент спільного редагування документів у форматі Markdown, розроблений командою Circus.

### Технологічний стек:
- Frontend
  - React + TS
  - Build tool: Bun
  - CodeMirror, Yjs
- Backend
  - Go
  - Gin, JWT
- Database
  - PostgreSQL
- Devops
  - Docker
  - GitHub Actions
  - Taskfile

### Features:
- Авторизація: Повністю реалізована авторизація за допомогою JWT + наявні усі необхідні перевірки на права на бекенді. 
- Керування документами:
  - Створення, видалення документів.
  - Перегляд списку документів відповідно до груп користувача.
- Керування групами:
  - Створення, оновлення, видалення груп.
  - Додавання, оновлення, видалення користувачів до груп, передача прав власника іншим користувачам.
- Редактор документів:
  - Сторінка редагування документів, з переглядом усіх користувачів що працюють над документом у цей момент.
  - Realtime візуалізація і рендер Markdown документу.
- Експорт документів:
  - Можливість експорту документів у форматі PDF, DOCX, HTML.
- Спільне редагування документів:
  - Можливість редагування документів в реальному часі одночасно для багатьох користувачів.
- Поширення документів через посилання:
  - Створення спеціальних посилань на документи, щоб навіть неавторизовані користувачі мали змогу переглядати документи в режимі реального часу.

### Infrastructure:
- CI
  - Автоматичний lint, тестування та докерізація на кожному PR реалізовне на GitHub Actions.
  - Покриття виноситься у файли `coverage.out` та `coverage.html`.
- Тестування
  - Статичний аналіз коду фронтенду та бекенду.
  - Unit тестування для більшості хендлерів.
  - Integration тестування.
- Докерізація
  - Автоматична докерізація на кожному PR реалізовне на GitHub Actions.

### Labs:
- Майже вся документація з лабораторних додана.
- Виконані всі, крім останньої, лабораторні роботи згідно з нашим баченням проєкту.

### Нереалізовані функції:
- Версіонування документів.
  - Причина: Відсутність часу на реалізацію цієї функції через брак правильного менеджменту.

### Висновки:
Проєкт Markdown Circus Docs успішно досяг своєї мети — створити функціональний інструмент для спільного редагування Markdown документів. Команда реалізувала усі критичні функції та забезпечила високу якість коду через автоматизацію CI/CD та тестування.
